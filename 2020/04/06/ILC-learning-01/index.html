变量、集合、约束、选择

  1. 集合划分
{p(X) ; q(X)} = 1 :- X = 1..n.

将数字1到n，划分进入两个集合之中。

  1. 局部变量与全局变量
仅仅出现在大括号内的变量是局部变量，否则是全局变量；
局部变量的规则实例仅仅是他自己一条；
全局变量的规则实例则是替换全部相关的变量值

例1：不使用局部变量，改写第二条规则

p(a; b). % 规则1
{q(X, Y) : p(X), p(Y)} = 1. % 规则2
%%%规则2改写为%%%
{q(a,a; a,b; b,a; b,b)} = 1.

例2：不使用局部变量，改写第三条规则

p(a; b). % 规则1
q(1..4). % 规则2
1 {r(X, Y) : p(X)} :- q(Y). % 规则3
%%%规则3改写为%%%
1{r(a,1; b,1)}.
1{r(a,2; b,2)}.
1{r(a,3; b,3)}.
1{r(a,4; b,4)}.

例3：涂色问题-每个点都需要涂一个颜色

vertex(1..5).
color(red;blue;green).
1{has_color(X, Y) : color(Y)}1 :- vertex(X).
  1. 关于约束

回答一个问题

:- not p.

和

p.

是否等价？

答案：不等价。前者表示不允许回答集中没有p，但不保证回答集有可能得到p。

极端反例：

{q}.
:- not p.

没有回答集（unsat）。

{q}.
p.

有两个回答集：{p,q} 与 {p}

例4：下一程序回答集？

{p(1..10)}.
:- p(2 * X).

实际是奇数枚举 — {p(1;3;5;7;9)}.

数值比较的约束可转非约束：

例如：同一个X，只具有唯一的f(X,Y)

:- f(X, Y1), f(X, Y2), Y1 != Y2.
%%%等价于%%%
Y1 = Y2 :- f(X, Y1), f(X, Y2).

如果一个规则的头部是一个数值比较，那么它其实不是一个定义，而是一个伪装的约束

Clingo中，匿名变量将被映射称为新谓词，例如p(X,_,Z)–>aux(X,Z)但这个谓词在回答集中是不会被看到的。

p :- not q.
q :- not p.

在prolog中会陷入无尽循环（infinite loop）

关于count的用法

p(1,a).
p(1,b).
p(1,c).
q(a;b).
r(X) :- X = #count{U,V : p(U,V), q(V)}

座位分配问题

问题描述

  • n个座位
  • 互相不喜欢的人不能连着坐
  • 互相喜欢的人坐在一起

个人解法

% prg: seats_arrange.lp
seat(1..n).
person(1..n).

% 一人一座
1{sit(P, S) : seat(S)}1 :- person(P).
P1 = P2 :- sit(P1,S1), sit(P2, S1).

% 喜欢坐一起的两种情况
|S1 - S2| = 1 |  |S1 - S2| = (n-1) :-  person(P1), person(P2), 
                                       likes(P1, P2), 
                                       sit(P1, S1), sit(P2, S2).
% 不喜欢不坐一起的两种情况之（1）
|S1 - S2| != 1 :-  person(P1), person(P2), 
                   dislikes(P1, P2), 
                   sit(P1, S1), sit(P2, S2).


% 不喜欢不坐一起的两种情况之（2）
|S1 - S2| != (n-1) :-  person(P1), person(P2), 
                       dislikes(P1, P2), 
                       sit(P1, S1), sit(P2, S2).

% 喜欢具有交换性（错误）
% likes(P2, P1) :- likes(P1, P2).
% dislikes(P2, P1) :- dislikes(P1, P2).

% 事实
likes(1,2; 3,4). 
dislikes(2,3; 1,3).

#show sit/2.

执行命令

clingo 0 -c n=6 seats_arrange.lp

需要注意的问题

关于一人一座的所有情况生成

% 一人一座
1{sit(P, S) : seat(S)}1 :- person(P).
P1 = P2 :- sit(P1,S1), sit(P2, S1).
% 1{sit(P, S) : person(P)}1 :- seat(S).

除了声明1个person有一个seat之外，还需要限定同一个座位只能有一个person做，即所谓有且仅有。

其实，这是一个相等空位的双射问题（bijection），1{sit(P, S) : seat(S)}1 :- person(P).是为人分配座位，如果按照相同的方式，考虑为座位分配人，应该写成1{sit(P, S) : person(P)}1 :- seat(S).使用这种写法，同样可以达到P1 = P2 :- sit(P1,S1), sit(P2, S1).的效果，双射问题应当对两个方向都进行描述。

对于邻座的考虑

邻座的定义实际上是一个简单的分段函数，本来想通过取余等方式使用一个式子定义，但是比较困难。通过定义adj/2，表达两个位置的相邻关系，程序会更简洁。

对于like/dislike的对称性考虑是否多余？

是的，而且可能是错误的。

考虑违背传递性的极端情况：

P喜欢Q，此时P和Q相邻，Q和P必然也相邻，但是我们依然不能说Q喜欢P，或不喜欢P。

扩展：如果有m个桌子

人数相应变为，只需要把sit/2扩展为sit/3，然后对每个桌子进行一次判定即可。

% prg: seats_arrange_ntable.lp
seat(1..n).
table(1..m).
person(1..n*m).

% （一人）<-->（一桌的一座）
1{sit(P, S, T) : seat(S), table(T)}1 :- person(P).
1{sit(P, S, T) : person(P)}1 :- seat(S), table(T).


% 喜欢坐一起的两种情况
|S1 - S2| = 1 |  |S1 - S2| = (n-1) :- person(P1),person(P2),likes(P1,P2), 
                                      sit(P1, S1, T), sit(P2, S2, T).

% 喜欢坐一起的要坐一桌
T1 = T2 :-  likes(P1, P2), sit(P1, S1, T1), sit(P2, S2, T2).
    																	 
% 不喜欢的人在一桌的时候，情况之（1）
|S1 - S2| != 1 :-  person(P1), person(P2), dislikes(P1, P2), 
                   sit(P1, S1, T), sit(P2, S2, T).


% 不喜欢的人在一桌的时候，情况之（2）
|S1 - S2| != (n-1) :-  person(P1), person(P2), 
                       dislikes(P1, P2), 
                       sit(P1, S1, T), sit(P2, S2, T).

% 事实
likes(1,2; 3,4). 
dislikes(2,3; 1,3).

#show sit/3.
